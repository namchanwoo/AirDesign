## 1. 개요

### 1.1 인텐트란?

**인텐트(Intent)**는 캐릭터가 그리드 위에서 **자동으로 실행할 행동의 의지**를 정의하는 시스템이다.

"이 캐릭터는 무엇을 하려고 하는가?"를 나타내며, 각 인텐트는 다음을 포함한다:

- **턴 주기**: 몇 턴마다 행동하는가
- **실행 조건**: 어떤 상황에서 행동이 성공하는가
- **실행 행동**: 조건 충족 시 무엇을 하는가
- **대기 행동**: 조건 불충족 시 무엇을 하는가

### 1.2 핵심 설계 원칙

| 원칙 | 설명 |
|------|------|
| **자동화** | 인텐트에 따라 캐릭터는 자동으로 행동한다 |
| **전략적 선택** | 상황에 맞는 인텐트 선택이 전투 결과에 영향 |
| **캐릭터 개성** | 고유 인텐트가 캐릭터의 정체성을 표현 |

### 1.3 용어 정의

| 용어 | 정의 |
|------|------|
| **인텐트 (Intent)** | 캐릭터의 자동 행동 의지를 정의하는 단위 |
| **고유 인텐트** | 특정 캐릭터만 사용할 수 있는 전용 인텐트 |
| **활성 인텐트** | 현재 전투에서 실행 중인 인텐트 |
| **인텐트 전환** | 현재 인텐트를 다른 인텐트로 즉시 변경하는 행위 (주기 리셋) |
| **턴 주기** | 인텐트 행동이 실행되는 턴 간격 |
| **턴 (Turn)** | 글로벌 모래시계가 뒤집어지는 단위 (약 3초, 조정 가능) |
| **턴 카운터** | 캐릭터 머리 위에 표시되는 남은 턴 수 (3→2→1→발동!) |
| **스피드** | 동시 발동 시 처리 순서를 결정하는 캐릭터 스탯 |
| **실행 조건** | 인텐트의 주요 행동이 발동하기 위한 조건 |
| **대기 행동** | 실행 조건 불충족 시 대신 실행되는 행동 |

## 2. 인텐트 슬롯 구조

### 2.1 슬롯 구성

각 캐릭터는 최대 **3개의 고유 인텐트 슬롯**을 보유한다.

```
┌─────────────────────────────────────────────────────────┐
│                   캐릭터 인텐트 슬롯                      │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  ┌─────────────┐  ┌─────────────┐  ┌─────────────┐     │
│  │ 고유 슬롯 1 │  │ 고유 슬롯 2 │  │ 고유 슬롯 3 │     │
│  │             │  │             │  │             │     │
│  │ (캐릭터    │  │ (캐릭터    │  │ (캐릭터    │     │
│  │  전용)     │  │  전용)     │  │  전용)     │     │
│  │             │  │             │  │             │     │
│  │ [해금: 초기]│  │ [해금: 조건]│  │ [해금: 조건]│     │
│  └─────────────┘  └─────────────┘  └─────────────┘     │
│                                                         │
└─────────────────────────────────────────────────────────┘

> 💡 **향후 확장 가능성**: 공통 인텐트 슬롯 추가 검토 중 (섹션 9.1 참조)
```

### 2.2 슬롯 특징

| 슬롯 유형 | 개수 | 장착 가능 인텐트 | 특징 |
|----------|------|-----------------|------|
| **고유 슬롯** | 1~3개 | 해당 캐릭터 전용 인텐트만 | 캐릭터 정체성, 강력한 효과 |

### 2.3 슬롯 해금

캐릭터는 **초기에 1개의 인텐트만** 보유한다.
추가 인텐트는 **육성 레벨 또는 조건**에 따라 해금된다.

#### 해금 구조 예시

```
[전사 A - 인텐트 해금 현황]

◆ 분노의 일격 (고유 1) ───────── 초기 해금
◇ 광폭화 (고유 2) ────────────── Lv.10에서 해금
◇ 최후의 저항 (고유 3) ───────── Lv.20에서 해금

◆ = 해금됨
◇ = 미해금
```

#### 해금 조건 유형

| 조건 유형 | 예시 |
|----------|------|
| **레벨 도달** | Lv.10, Lv.15, Lv.20 등 |
| **스토리 진행** | 특정 챕터 클리어 |
| **업적 달성** | "100회 전투 승리" 등 |
| **아이템 사용** | 인텐트 해금 아이템 |

> ⚠️ **[검토 필요]** 구체적인 해금 조건 및 레벨은 밸런싱 단계에서 확정

## 3. 인텐트 행동 시스템

### 3.1 행동 구조

모든 인텐트는 **실행 조건**과 **두 가지 행동**을 가진다.

```
┌─────────────────────────────────────────────────────────┐
│                    인텐트 행동 구조                       │
├─────────────────────────────────────────────────────────┤
│                                                         │
│                   [행동 주기 도달]                       │
│                         │                               │
│                         ▼                               │
│                   ┌───────────┐                         │
│                   │ 실행 조건 │                         │
│                   │   체크    │                         │
│                   └─────┬─────┘                         │
│                         │                               │
│            ┌────────────┴────────────┐                  │
│            │                         │                  │
│            ▼                         ▼                  │
│     [조건 충족]                [조건 불충족]             │
│            │                         │                  │
│            ▼                         ▼                  │
│   ┌────────────────┐       ┌────────────────┐          │
│   │   실행 행동    │       │   대기 행동    │          │
│   │               │       │               │          │
│   │ (주요 효과)   │       │ (보조 효과)   │          │
│   └────────────────┘       └────────────────┘          │
│            │                         │                  │
│            └────────────┬────────────┘                  │
│                         │                               │
│                         ▼                               │
│              [다음 행동 주기 시작]                       │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

### 3.2 실행 조건 유형

| 조건 유형 | 설명 | 예시 |
|----------|------|------|
| **같은 행 적 존재** | 같은 행에 적이 있어야 함 | 공격 계열 인텐트 |
| **같은 행 아군 존재** | 같은 행에 아군이 있어야 함 | 지원 계열 인텐트 |
| **아군 존재** | 어디든 아군이 있어야 함 | 힐 계열 인텐트 |
| **HP 조건** | 특정 HP 비율 이상/이하 | 위기 대응 인텐트 |
| **무조건** | 항상 실행 | 자가 버프 인텐트 |

### 3.3 실행/대기 행동 예시

#### 예시 1: 공격형 인텐트

```
인텐트: "분노의 일격"
턴 주기: 3턴

[실행 조건] 같은 행에 적이 존재함
    │
    ├─ 충족 → [실행 행동] 같은 행 적 1명에게 5 데미지
    │
    └─ 불충족 → [대기 행동] 코스트 +1
```

#### 예시 2: 힐러형 인텐트

```
인텐트: "치유의 손길"
턴 주기: 2턴

[실행 조건] HP가 가장 낮은 아군이 존재함
    │
    ├─ 충족 → [실행 행동] 해당 아군 HP 5 회복
    │
    └─ 불충족 → [대기 행동] 자신 HP 2 회복
```

#### 예시 3: 방어형 인텐트

```
인텐트: "철벽 수호"
턴 주기: 2턴

[실행 조건] 같은 행에 아군이 존재함
    │
    ├─ 충족 → [실행 행동] 같은 행 아군 받는 피해 -30% (2턴)
    │
    └─ 불충족 → [대기 행동] 자신에게 보호막 3 부여
```

### 3.4 대기 행동의 역할

대기 행동은 **실행 조건 불충족 시에도 행동 주기가 헛되지 않게** 하는 보조 행동이다.

| 대기 행동 유형 | 설명 | 예시 |
|---------------|------|------|
| **코스트 충전** | 카드 사용을 위한 자원 확보 | 코스트 +1, +2 |
| **자가 버프** | 다음 행동 준비 | 다음 공격력 +20% |
| **자가 회복** | 생존력 확보 | 자신 HP 2 회복 |
| **보호막** | 방어 준비 | 자신에게 보호막 부여 |

### 3.5 전투 시작 시 규칙

> ✅ **확정됨** (v2.0 기준)

#### 3.5.1 초기 인텐트 결정

| 항목 | 규칙 |
|------|------|
| **초기 인텐트** | 파티 설정 화면에서 전투 전 미리 지정 |
| **미지정 시** | 첫 번째 슬롯 인텐트 (기본 고유 인텐트) |
| **전투 간 기억** | 저장하지 않음 (매 전투 파티 설정 기준) |

#### 3.5.2 초기 턴 카운터

| 항목 | 규칙 |
|------|------|
| **시작 값** | **최대 주기**로 시작 |
| **예시** | 3턴 주기 인텐트 → 전투 시작 시 카운터 = 3 |
| **적용 대상** | 아군 + 적군 모두 동일 |

```
[전투 시작 예시]

아군 전사 (분노의 일격, 3턴 주기)
┌────────┐
│ 전사 A │
│   ⚔️   │
│   [3]  │  ← 최대 주기로 시작
└────────┘

적 고블린 (난폭한 공격, 2턴 주기)
┌────────┐
│ 고블린 │
│   ⚔️   │
│   [2]  │  ← 최대 주기로 시작
└────────┘

→ 첫 턴 종료 시: 전사 [2], 고블린 [1]
→ 두 번째 턴 종료 시: 전사 [1], 고블린 발동!
```

### 3.6 동시 발동 규칙

> ⚠️ **프로토타입** (v2.1 기준) - 추후 기획 변경 예정

동일 턴에 여러 유닛의 인텐트가 발동할 경우, **스피드(Speed) 스탯**으로 처리 순서를 결정한다.

#### 3.6.1 처리 순서

| 우선순위 | 기준 |
|---------|------|
| 1순위 | **스피드 수치** 높은 순 |
| 동률 시 | **랜덤** |

> 💡 **프로토타입 노트**: 현재는 동률 시 랜덤 처리. 추후 플레이테스트 결과에 따라
> 아군 우선, 슬롯 순서 등 다른 규칙으로 변경될 수 있음.

#### 3.6.2 동시 발동 예시

```
[상황] 같은 턴에 3명의 인텐트 발동

전사 A (스피드: 50)  - 분노의 일격
힐러 B (스피드: 70)  - 치유의 손길
적 오크 (스피드: 60) - 광역 강타

처리 순서:
1. 힐러 B (스피드 70) → 아군 치유
2. 적 오크 (스피드 60) → 광역 공격
3. 전사 A (스피드 50) → 단일 공격

※ 만약 힐러 B와 적 오크의 스피드가 동일(60)했다면 → 랜덤 결정
```

#### 3.6.3 전략적 의미

- **높은 스피드** = 선제 행동으로 전투 주도권
- **힐러 스피드** = 아군 피격 전 선힐 vs 피격 후 후힐
- **딜러 스피드** = 적 행동 전 처치로 피해 방지
- **버프/디버프 타이밍** = 순서에 따라 효과 적용 여부 결정
- **동률 시 불확실성** = 스피드 동률 상황은 예측 불가 (랜덤)

### 3.8 실행/대기 행동 밸런스 가이드라인

> ✅ **확정됨** (v1.4 기준)
>
> **중요도: HIGH** - 핵심 재미에 직결되는 밸런스 가이드라인
>
> **핵심 관점**: 경직된 규칙이 아닌 **유연한 가이드라인**. 캐릭터/인텐트별 개별 설계 우선.

#### 3.8.1 설계 철학

**기본 관점:**
- 대기 행동 ≠ 손해, ≠ 약함
- 상황과 전략에 따라 **대기가 더 나은 선택일 수 있음**
- 실행/대기 모두 **의미 있는 선택지**로 설계

**피해야 할 패턴:**

| 패턴 | 문제 | 수정 방향 |
|------|------|----------|
| **역전 설계** | 대기 > 실행 수치 | 동일 효과 유형이면 실행 ≥ 대기 |
| **대기 = 페널티** | 조건 불충족 = 손해 인식 | 대기도 전략적 가치 부여 |

#### 3.8.2 밸런스 가이드라인

> 아래는 **참고 방향**이며, 캐릭터/인텐트 컨셉에 따라 **유연하게 적용**

**방향성 참고 (권장, 필수 아님):**

| 행동 | 경향성 | 설명 |
|------|--------|------|
| **실행 행동** | 외부 효과 경향 | 적/아군에게 영향 (데미지, 버프, 디버프) |
| **대기 행동** | 내부 효과 경향 | 자신에게 영향 (자힐, 코스트, 축적) |

**유연성 원칙:**
- 모든 인텐트가 위 패턴을 따를 필요 없음
- 캐릭터 컨셉과 역할에 맞는 **개별 설계 우선**
- 대기 행동이 외부 효과여도 무방 (컨셉에 맞다면)

**예시 - 전략적 선택 상황:**
```
탱커 - 철벽 수호
- 실행 조건: 같은 행에 아군 존재
- 실행 행동: 해당 아군 피해 감소 30%
- 대기 행동: 자신 보호막 3

→ 아군 보호 급하면 → 실행 조건 맞추기 (이동/배치)
→ 자신이 위험하면 → 대기로 보호막 축적이 나을 수도
→ 플레이어의 상황 판단에 따른 전략적 선택
```

#### 3.8.3 고유 자원 시스템 (선택적)

**개념:**
- **일부 캐릭터에만** 적용되는 개성 표현 요소
- **모든 캐릭터 필수 아님** - 단순한 캐릭터도 가치 있음

**적용 예시:**

| 캐릭터 | 고유 자원 | 축적 방법 | 효과 |
|--------|----------|----------|------|
| 뱀파이어 | 갈증 | 흡혈 공격 시 | 능력 강화, 특수 스킬 해금 |
| 버서커 | 분노 | 피격 시 / 대기 행동 | 공격력 증가, 방어력 감소 |
| 마법사 | 집중 | 시전 준비 (대기) | 마법 위력/범위 증가 |
| 일반 전사 | **없음** | - | 단순 명료한 플레이 |

**스택 연동 인텐트 예시:**
```
버서커 - 분노의 일격
- 실행 조건: 같은 행 적 존재
- 실행 행동: 적 1명에게 5 + (분노 × 2) 데미지, 분노 소모
- 대기 행동: 분노 +1 (최대 5)

→ 분노 0: 즉시 약한 공격 (5뎀)
→ 분노 5: 강력한 일격 (15뎀)
→ "지금 칠 것인가 vs 축적 후 강하게 칠 것인가" 전략
```

#### 3.8.4 설계 시 체크리스트

| 항목 | 확인 내용 |
|------|----------|
| **역전 확인** | 동일 효과 유형에서 대기 > 실행 아닌가? |
| **전략적 가치** | 대기 행동도 상황에 따라 선택할 이유가 있는가? |
| **캐릭터 컨셉** | 인텐트가 캐릭터 정체성/역할에 맞는가? |
| **개별 밸런싱** | 가이드라인에 맞추려고 억지 설계하지 않았는가? |

#### 3.8.5 효과 가치 환산표 (참고용)

> 밸런싱 시 **참고 기준**. 절대적 수치 아님.

| 효과 종류 | 참고 가치 | 비고 |
|----------|:---------:|------|
| 데미지 1 | 1.0 | 기준 |
| HP 회복 1 | 1.0~1.2 | 생존 가치 고려 |
| 보호막 1 | 0.8~1.0 | 일회성, 선제 방어 |
| 코스트 1 | 1.5~2.0 | 카드 활용 가치 |
| 버프 10% (4초) | 1.0~1.5 | 지속시간/효과에 따라 |

## 4. 전투 중 인텐트 운용

### 4.1 활성 인텐트

전투 중에는 **오직 1개의 인텐트만 활성화**된다.

```
[전투 중 상태]

보유 인텐트: 고유1, 고유2, 고유3
                ↓
         현재 활성: 고유2
                ↓
    고유2의 의지대로만 행동함
```

### 4.2 인텐트 전환 규칙

> ⚠️ **프로토타입** (v2.1 기준) - 추후 기획 변경 예정

#### 4.2.1 기본 규칙

| 규칙 | 내용 |
|------|------|
| **전환 비용** | 없음 (자유롭게 전환 가능) |
| **전환 방식** | **즉시 전환** (아이콘 즉시 변경) |
| **전환 턴 상태** | 전환한 턴에서는 **Wait/X** 표시 (행동 없음) |
| **적용 시점** | **다음 턴부터** 최대 주기로 시작 |
| **이전 인텐트** | 남은 턴 무관하게 즉시 취소됨 |
| **카드 사용 중** | 전환 불가, 카드 완료 후 가능 |

#### 4.2.2 전환 흐름

```
┌─────────────────────────────────────────────────────────────┐
│                    인텐트 전환 프로세스 (턴 기반)              │
└─────────────────────────────────────────────────────────────┘

[상황] 현재 인텐트: "분노의 일격" (3턴 주기), 현재 카운터 [2]

턴 진행 ──────────────────────────────────────────────────→

  턴 N          턴 N+1         턴 N+2         턴 N+3
    │              │              │              │
    ▼              │              │              │
 분노의 일격       │              │              │
   [2]            │              │              │
    │              │              │              │
    │ 플레이어가   │              │              │
    │ "광폭화"로   │              │              │
    │ 전환 선택    │              │              │
    │              │              │              │
    ▼              │              │              │
┌────────────┐     │              │              │
│ 즉시 전환! │     │              │              │
│            │     │              │              │
│ "분노의    │     │              │              │
│  일격" 취소│     │              │              │
│            │     │              │              │
│ "광폭화"   │     │              │              │
│  [X]       │     ▼              ▼              ▼
│ (대기 중)  │   [2]            [1]           발동!
└────────────┘  (최대 주기로    (카운트       (행동 실행)
                 시작)          다운)
```

**핵심 포인트:**
- A 인텐트에서 B 인텐트로 변경 시, A가 몇 턴 남았든 **즉시 취소**
- 전환한 턴에서는 B 인텐트가 **[X] 또는 Wait** 상태로 표시 (행동 없음)
- **다음 턴부터** B 인텐트의 최대 주기로 시작 (예: 2턴 주기 → [2]부터 시작)
- 자주 전환하면 **행동 실행 빈도가 낮아짐** (전환 턴 손실)

#### 4.2.3 전환 상태 표시

```
[전환 직후 - 해당 턴]

        💥        ← 광폭화 아이콘 (새 인텐트)
       [X]        ← 대기 상태 (이번 턴 행동 없음)
     ┌─────┐
     │전사A│
     └─────┘

※ [X] 또는 [Wait] 또는 [⏸] 등으로 표시 (UI 디자인에 따라 결정)
```

```
[다음 턴부터 - 정상 작동]

        💥        ← 광폭화 아이콘
       [2]        ← 최대 주기로 시작 (2턴 주기 인텐트)
     ┌─────┐
     │전사A│
     └─────┘
```

#### 4.2.4 전환 비용

> ⚠️ **프로토타입** (v2.1 기준)

**결정: 추가 비용 없음 (무료 전환 + 1턴 대기 + 주기 리셋)**

| 항목 | 내용 |
|------|------|
| **전환 비용** | 없음 |
| **암묵적 비용** | 전환 턴 행동 불가 + 다음 턴 최대 주기로 리셋 |
| **재전환 제한** | 없음 (언제든 전환 가능) |

**선택 근거:**

| 근거 | 설명 |
|------|------|
| **1턴 대기 = 전환 비용** | 전환 시 해당 턴 행동 손실로 신중한 선택 유도 |
| **단순함 유지** | 복잡한 비용 계산 불필요, 직관적 |
| **빠른 대응** | 긴급 상황에 즉각 전환 가능 (다음 턴부터 효과) |
| **기획 의도 부합** | 조작 스트레스 감소, 미드코어 타겟 |

**전략적 깊이 확보:**
```
전환 타이밍 자체가 전략적 선택

예시 1 - 전환 손해 케이스:
- 현재 인텐트 "분노의 일격" [1] → 다음 턴 발동 예정
- 지금 전환 시 → 이번 턴 [X] + 다음 턴 새 인텐트 [2]
- 결과: 원래 1턴 후 행동할 것이 3턴 후로 지연

예시 2 - 전환 이득 케이스:
- 현재 인텐트 "분노의 일격" [3] → 3턴 후 발동 예정
- 지금 "광폭화"(2턴 주기)로 전환 시 → 이번 턴 [X] + 다음 턴 [2]
- 결과: 3턴 후 → 2턴 후로 앞당겨짐

→ "지금 전환할 것인가 vs 현재 인텐트 발동 후 전환할 것인가" 판단
→ 전환 턴 손실을 감안한 전략적 의사결정
```

- 인텐트 주기(4~6초) 동안 플레이어의 인터랙션이 제한적
- 대기 시간 중 "방관자 느낌" 우려

**검토 필요 항목:**
- 긴급 전환 메커니즘 (코스트 소모하여 행동 주기 단축?)
- 행동 주기 영향 버프/디버프 (가속/둔화)
- 카드 사용 외 추가 인터랙션 요소
- 플레이어 개입 빈도와 자동 전투 밸런스

## 5. 고유 인텐트

### 5.1 개념

**고유 인텐트**는 특정 캐릭터만 사용할 수 있는 **전용 인텐트**이다.

- 캐릭터의 **서사, 정체성, 역할**을 반영
- **강력하고 특화된** 효과
- 캐릭터당 **최대 3개** 보유

### 5.2 고유 인텐트 설계 원칙

| 원칙 | 설명 |
|------|------|
| **역할 표현** | 캐릭터의 전투 역할(딜러/탱커/힐러 등)을 명확히 |
| **서사 연결** | 캐릭터의 배경/성격이 인텐트에 반영 |
| **다양성** | 3개의 인텐트가 서로 다른 상황에 유용하도록 |
| **전환 유도** | 인텐트 간 전환을 통한 전략적 운용 |

### 5.3 고유 인텐트 구조

각 고유 인텐트는 다음 요소를 정의한다:

```
┌─────────────────────────────────────────────────────────┐
│ 고유 인텐트: [인텐트 이름]                               │
├─────────────────────────────────────────────────────────┤
│                                                         │
│ ▶ 기본 정보                                             │
│   - 소속 캐릭터: [캐릭터명]                              │
│   - 턴 주기: [X턴]                                      │
│   - 아이콘: [아이콘 설명]                               │
│   - 해금 조건: [조건]                                   │
│                                                         │
│ ▶ 실행 조건                                             │
│   - [조건 설명]                                         │
│                                                         │
│ ▶ 실행 행동 (조건 충족 시)                              │
│   - 타겟: [타겟팅 규칙]                                 │
│   - 효과: [데미지, 버프, 디버프 등]                     │
│                                                         │
│ ▶ 대기 행동 (조건 불충족 시)                            │
│   - 효과: [코스트 충전, 자힐, 버프 등]                  │
│                                                         │
│ ▶ 특수 효과 (선택적)                                    │
│   - [고유 자원 연동, 조건부 효과 등]                    │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

### 5.4 고유 인텐트 예시

#### 예시: 전사 캐릭터

| 인텐트 | 주기 | 실행 조건 | 실행 행동 | 대기 행동 |
|--------|------|----------|----------|----------|
| 분노의 일격 | 3턴 | 같은 행 적 존재 | 적 1명 5데미지 | 코스트 +1 |
| 광폭화 | 2턴 | 같은 행 적 존재 | 적 2명 각 3데미지 | 다음 공격력 +20% |
| 최후의 저항 | 4턴 | 같은 행 적 존재 | 적 1명 4데미지 + 자힐 3 | HP 5% 회복 |

## 6. 적 유닛의 인텐트

### 6.1 기본 원칙

적 유닛도 **동일한 인텐트 시스템**을 사용한다.

- 행동 주기에 따른 자동 행동
- 실행 조건 / 대기 행동 구분
- 인텐트에 따른 행동 패턴

### 6.2 적 유형별 인텐트 복잡도

| 적 유형 | 복잡도 | 설명 |
|--------|--------|------|
| **잡몹** | 단순 | 1개의 간단한 인텐트, 예측 쉬움 |
| **정예몹** | 중간 | 2~3개 인텐트, 전환 패턴 있음 |
| **보스몹** | 복잡 | 다수 인텐트, 페이즈별 전환, 특수 메커니즘 |

### 6.3 적 인텐트 UI 표시

플레이어는 **적의 현재 인텐트와 남은 시간을 볼 수 있다**.

#### 6.3.1 기본 표시 (적 머리 위)

```
        [적 유닛]
           │
    ┌──────┴──────┐
    │    ⚔️       │ ← 인텐트 아이콘
    │    [2]      │ ← 남은 턴 카운터
    └─────────────┘
           │
           ▼
      [클릭 시 상세 팝업]
```

각 인텐트는 **고유 아이콘**을 가진다:
- ⚔️ 공격 계열
- 🛡️ 방어 계열
- 💚 회복 계열
- ⚡ 버프/디버프 계열
- 등등 (인텐트별 개별 아이콘)

#### 6.3.2 상세 정보 팝업 (아이콘 클릭 시)

```
┌────────────────────────────────────┐
│ 난폭한 휘두르기                     │
├────────────────────────────────────┤
│                                    │
│ 턴 주기: 2턴                       │
│ 남은 턴: 1                         │
│                                    │
│ ──────────────────────────────    │
│                                    │
│ [실행 행동]                        │
│ 같은 행 전체 적에게 3 데미지       │
│                                    │
│ [조건 불충족 시]                   │
│ 공격력 +10% 버프 (2턴)             │
│                                    │
└────────────────────────────────────┘
```

#### 6.3.3 전체 적 인텐트 표시 예시

```
[전투 화면 - 적 진영]

┌────────┐  ┌────────┐  ┌────────┐
│ 고블린 │  │ 고블린 │  │ 오크   │
│ 전사   │  │ 궁수   │  │ 대장   │
│        │  │        │  │        │
│  ⚔️    │  │  🏹    │  │  💀    │
│  [2]   │  │  [1]   │  │  [3]   │
└────────┘  └────────┘  └────────┘
```

### 6.4 전략적 의미

적 인텐트가 표시되면 플레이어는:

1. **적 행동 예측**: 언제, 무엇을 할지 알 수 있음
2. **인텐트 대응**: 적 인텐트에 맞춰 아군 인텐트 조정
3. **타이밍 플레이**: 적 행동 직전/직후에 카드 사용
4. **우선순위 판단**: 위험한 인텐트의 적 먼저 처치

**예시 전략:**
```
상황: 오크 대장의 "광역 강타" 인텐트가 1초 후 발동

대응:
1. 힐러의 인텐트를 "보호막 부여"로 전환 예약
2. 탱커를 앞줄로 이동시키는 카드 사용
3. 적 행동 후 딜러의 공격 카드로 반격
```

### 6.5 적 정보 단계적 공개 (검토 필요)

> ⚠️ **[내용추가예정]** 적 인텐트 정보 공개 수준 논의 필요
>
> **중요도: MEDIUM** - 숙련도 기반 진행감에 영향

**현재 문제점:**
- 모든 적 정보가 처음부터 공개 → 탐색의 재미 부족
- 처음 만난 적도 완벽히 파악 가능 → 긴장감 저하

#### 6.5.1 공개 수준 옵션 (검토 중)

| 옵션 | 설명 | 장점 | 단점 |
|------|------|------|------|
| **A. 완전 공개** | 모든 정보 즉시 공개 | 전략 수립 용이 | 탐색 재미 없음 |
| **B. 처치 후 공개** | 최초 처치 시 도감 등록 | 탐색 동기 | 첫 조우 불리 |
| **C. 점진적 공개** | 조우 횟수에 따라 정보 추가 | 성장감 | 시스템 복잡 |
| **D. 조사 스킬** | 특정 능력/카드로 정보 확인 | 전략 요소 | 필수 슬롯화 위험 |

#### 6.5.2 점진적 공개 단계 (검토 중)

> **[내용추가예정]** 채택 여부 논의 필요

```
[개념]
적과의 조우/처치 횟수에 따라 정보 공개 레벨 상승

1단계 (최초 조우): 아이콘 + 이름 + 행동 유형 (공격/방어/지원)
2단계 (1회 처치): 행동 주기 + 실행 조건
3단계 (3회 처치): 실행/대기 행동 상세 수치
4단계 (10회 처치): 전환 패턴 + 특수 행동 정보

[효과]
- 반복 플레이 동기 부여
- 도감 수집 콘텐츠화
```

#### 6.5.3 검토 필요 항목

- 최초 조우 시 최소 공개 정보 범위
- 정보 공개 진행 저장 방식 (계정? 캐릭터?)
- 보스급 적 예외 처리 여부
- 정보 미공개로 인한 불공정 체감 최소화 방안

## 7. UI/UX 설계

### 7.1 인텐트 변경 UI (오버헤드 버튼 방식)

> ✅ **확정됨** (v2.1 기준) - PC 싱글플레이어 플랫폼

#### 7.1.1 핵심 컨셉

**"전투 화면을 유지하면서 최소 클릭으로 인텐트 변경"**

기존 팝업 방식의 문제점(전투 화면 가림, 3단계 이상 클릭)을 해결하기 위해
캐릭터 머리 위 인텐트 아이콘을 직접 클릭하는 **오버헤드 버튼 방식** 채택.

#### 7.1.2 기본 흐름 (2단계)

```
1. 캐릭터 머리 위 인텐트 아이콘 클릭
              ↓
2. 가로로 펼쳐진 인텐트 목록에서 원하는 인텐트 클릭
              ↓
      즉시 전환 완료 (새 주기 시작)
```

**장점:**
- 2클릭으로 완료 (기존 3~4클릭 → 2클릭)
- 전투 화면을 가리지 않음 (전체 상황 파악 유지)
- 직관적 (변경하려는 대상을 직접 클릭)

#### 7.1.3 그리드 영역 인텐트 표시

```
[그리드 영역 - 기본 상태]

     [아군 진영]                              [적 진영]

      ⚔️[2]        💚[1]                        💀[3]        🔥[2]
     ┌─────┐      ┌─────┐                     ┌─────┐      ┌─────┐
     │전사A│      │힐러B│         VS          │적1  │      │적2  │
     └─────┘      └─────┘                     └─────┘      └─────┘

      🛡️[1]        🔮[2]                        ⚡[1]        🗡️[4]
     ┌─────┐      ┌─────┐                     ┌─────┐      ┌─────┐
     │탱커C│      │마법D│                     │적3  │      │적4  │
     └─────┘      └─────┘                     └─────┘      └─────┘

⚔️[2] = 인텐트 아이콘 + 남은 턴 카운터 (아군만 클릭 가능)
```

#### 7.1.4 아군 인텐트 클릭 시 (목록 펼침)

```
                     [인텐트 목록 - 가로 펼침]
                 ┌─────┬─────┬─────┐
                 │ ⚔️  │ 💥  │ 🛡️  │
                 │ [3] │ [2] │ [4] │
                 │ ●  │     │     │
                 └──┬──┴─────┴─────┘
                    │
      ⚔️[2] ────────┘ (현재 활성 = ● 표시)
     ┌─────┐
     │전사A│  ← 선택된 캐릭터 하이라이트
     └─────┘

● = 현재 활성 인텐트
[숫자] = 해당 인텐트의 행동 주기
```

**상호작용:**
- 목록의 다른 인텐트 클릭 → 즉시 해당 인텐트로 전환
- 현재 활성 인텐트(●) 클릭 → 목록 닫힘 (변경 없음)
- 목록 외부 클릭 → 목록 닫힘 (변경 없음)
- ESC 키 → 목록 닫힘

#### 7.1.5 아군 vs 적 시각적 구분

| 요소 | 아군 (클릭 가능) | 적 (읽기 전용) |
|------|------------------|----------------|
| **아이콘 프레임** | 둥근 모서리 + 파란/초록 테두리 | 날카로운 모서리 + 빨강/보라 테두리 |
| **호버 효과** | 밝아짐 + 커서 변경 | 없음 (정보 표시만) |
| **클릭 반응** | 목록 펼침 | 인텐트 정보 툴팁만 표시 |
| **턴 카운터** | 파란색 숫자 | 빨간색 숫자 |

#### 7.1.6 인텐트 변경 피드백

```
[인텐트 전환 시 시각 피드백]

1. 선택한 인텐트 아이콘이 확대되며 반짝임
2. 기존 아이콘 → 새 아이콘으로 전환 애니메이션
3. "찰칵" 효과음 재생
4. 턴 카운터가 최대값으로 리셋되는 애니메이션
5. 캐릭터에 버프 획득 이펙트 (간단한 파티클)
```

### 7.2 인텐트 상태 표시

#### 7.2.1 오버헤드 인텐트 표시 (기본 상태)

캐릭터 머리 위에 현재 인텐트 아이콘과 턴 카운터를 표시한다.

```
        ⚔️        ← 인텐트 아이콘 (아군은 클릭 가능)
       [2]        ← 남은 턴 카운터
     ┌─────┐
     │전사A│
     └─────┘
```

#### 7.2.2 오버헤드 표시 상세 스펙

| 요소 | 설명 |
|------|------|
| **아이콘 크기** | 캐릭터 그리드 셀 너비의 약 40% |
| **턴 카운터 위치** | 아이콘 아래, 캐릭터 위 |
| **턴 카운터 색상** | 아군=파랑, 적=빨강 |
| **발동 임박 표시** | 카운터 [1] 시 아이콘 깜빡임 |
| **발동 순간** | 아이콘 확대 + 플래시 효과 |

#### 7.2.3 턴 카운터 시각화

```
[턴 카운터 상태별 표시]

[3] = 일반 상태 (3턴 남음)
[2] = 일반 상태 (2턴 남음)
[1] = 경고 상태 (깜빡임, 색상 강조) ← 다음 턴 발동!
[!] = 발동 중 (아이콘 확대 + 이펙트)
```

#### 7.2.4 마우스 호버 시 상세 정보

아군/적 인텐트 아이콘에 마우스 호버 시 툴팁으로 상세 정보 표시.

### 7.3 조건 충족 표시 시스템 (검토 필요)

> ⚠️ **[내용추가예정]** 조건 충족 여부 시각적 피드백 논의 필요
>
> **중요도: HIGH** - 인지 부하 감소 및 직관적 이해에 직결

**현재 문제점:**
- 각 유닛의 실행 조건 충족 여부 파악 어려움
- "이 유닛이 왜 대기 행동을 했지?" 혼란
- 조건 확인을 위해 팝업을 일일이 열어야 함

#### 7.3.1 표시 방식 옵션 (검토 중)

| 옵션 | 설명 | 장점 | 단점 |
|------|------|------|------|
| **A. 아이콘 색상** | 충족=밝음, 불충족=어두움 | 직관적 | 색맹 고려 필요 |
| **B. 테두리 효과** | 조건 충족 시 테두리 강조 | 명확함 | 화면 복잡 |
| **C. 예측 표시선** | 실행 행동 대상에 점선 연결 | 대상 파악 용이 | 다수 유닛 시 혼란 |
| **D. 상태 텍스트** | "대기 중", "준비 완료" 텍스트 | 명시적 | 공간 차지 |

#### 7.3.2 검토 필요 항목

- 조건 충족/불충족 시 시각적 구분 방식
- 다수 유닛 동시 표시 시 가시성 확보 방안
- 조건 변화 시 실시간 업데이트 빈도
- 접근성 고려 (색맹 대응)

### 7.4 위협 우선순위 시스템 (검토 필요)

> ⚠️ **[내용추가예정]** 적 인텐트 위협도 표시 논의 필요
>
> **중요도: MEDIUM** - 다수 적 상황에서 의사결정 지원

**현재 문제점:**
- 다수 적 존재 시 어느 적을 주시해야 할지 불명확
- 위험한 인텐트 발동 직전 인지 실패
- 긴급 대응 필요 상황 파악 어려움

#### 7.4.1 우선순위 표시 옵션 (검토 중)

| 옵션 | 설명 | 장점 | 단점 |
|------|------|------|------|
| **A. 위협 등급 색상** | 높음=빨강, 중간=노랑, 낮음=초록 | 직관적 | 색상 과다 |
| **B. 경고 아이콘** | 고위협 인텐트에 경고 표시 | 명확함 | 기준 설정 필요 |
| **C. 사운드 알림** | 고위협 인텐트 발동 임박 시 효과음 | 시각 부담 감소 | 과다 시 소음 |
| **D. 화면 테두리** | 위협 방향에 테두리 효과 | 방향 인식 | 익숙해지면 무시 |

#### 7.4.2 위협 등급 기준 (검토 중)

> **[내용추가예정]** 위협 등급 산정 기준 논의 필요

```
[위협도 산정 요소 후보]
- 예상 데미지량
- 광역 범위
- 디버프 심각도
- 발동까지 남은 시간
- 아군 현재 HP 상태
```

#### 7.4.3 검토 필요 항목

- 위협 등급 구분 개수 (2단계? 3단계?)
- 자동 계산 vs 인텐트별 수동 설정
- 위협도 계산 공식 (데미지 x 범위 / 남은시간?)
- 알림 빈도 조절 (과다 알림 방지)

## 8. 밸런스 가이드라인

### 8.1 턴 주기 기준

| 주기 | 특성 | 적합한 인텐트 유형 |
|------|------|-------------------|
| 1턴 | 매우 빠름 | 약한 효과, 빠른 축적 |
| 2턴 | 빠름 | 기본 공격, 경미한 효과 |
| 3턴 | 보통 | 표준 인텐트 |
| 4턴 | 느림 | 강한 효과 |
| 5턴+ | 매우 느림 | 매우 강한 효과 |

### 8.2 인텐트 간 트레이드오프

각 인텐트는 **장단점**이 있어야 한다:

| 예시 인텐트 | 장점 | 단점 |
|------------|------|------|
| 강타형 | 높은 단일 데미지 | 광역 불가 |
| 광역형 | 다수 공격 | 낮은 개별 데미지 |
| 자힐형 | 생존력 | 낮은 공격력 |
| 충전형 | 빠른 코스트 | 행동 기여 없음 |

## 9. 향후 확장 가능성

### 9.1 공통 인텐트 시스템

> ⚠️ **추후 기획 추가 예정**
>
> **개념:** 모든(또는 대부분) 캐릭터가 장착 가능한 범용 인텐트
>
> **검토 예정 항목:**
> - 공통 인텐트 목록 및 효과
> - 고유 인텐트 대비 밸런스 (60~80% 수치)
> - 공통 슬롯 해금 조건
> - 캐릭터 역할/속성별 장착 제한
> - "선택적 조율" 개념 (단점 보완/장점 강화)

### 9.2 인텐트 강화 시스템

> ⚠️ **추후 추가 예정 / 미정**
> 기본 인텐트 시스템 안정화 및 테스트 후 검토

| 아이디어 | 설명 |
|----------|------|
| **숙련도** | 같은 인텐트 반복 사용 시 효과 강화 |
| **강화 재료** | 아이템으로 인텐트 효과 업그레이드 |

### 9.3 전장 효과 시스템 (검토 필요)

> ⚠️ **[내용추가예정]** 인텐트와 연동되는 전장 효과 논의 필요
>
> **중요도: MEDIUM** - 로그라이트 런별 다양성에 기여

**개념:**
- 런마다 등장하는 전장 특수 효과
- 특정 인텐트 계열 강화/약화
- 전략적 인텐트 선택에 영향

#### 9.3.1 전장 효과 예시 (검토 중)

> **[내용추가예정]** 채택 여부 논의 필요

```
[전장 효과 후보]

분노의 전장:
- 공격 계열 인텐트 +20% 데미지
- 방어 계열 인텐트 -20% 효과

냉기의 전장:
- 모든 인텐트 행동 주기 +1초
- 대기 행동 효과 +30%

혼돈의 전장:
- 3턴마다 무작위 인텐트 강제 전환
- 전환 시 주기 50% 유지
```

#### 9.3.2 검토 필요 항목

- 전장 효과 등장 빈도 (모든 전투? 특수 구간만?)
- 전장 효과 중첩 가능 여부
- 보스 스테이지 전용 전장 효과
- 런 시작 시 전장 효과 미리보기 여부

### 9.4 인텐트 시너지 시스템 (검토 필요)

> ⚠️ **[내용추가예정]** 파티 구성 기반 시너지 논의 필요
>
> **중요도: MEDIUM** - 파티 빌딩 깊이 확보

**개념:**
- 특정 인텐트 조합 시 추가 효과 발동
- 파티 구성 전략에 의미 부여
- 로그라이트 빌드 다양성

#### 9.4.1 시너지 유형 (검토 중)

> **[내용추가예정]** 채택 여부 논의 필요

| 시너지 유형 | 조건 | 효과 예시 |
|-------------|------|----------|
| **동기화** | 같은 계열 인텐트 2개 이상 활성 | 해당 계열 효과 +15% |
| **연계** | A 인텐트 발동 후 B 인텐트 발동 | 보너스 효과 추가 |
| **공명** | 같은 행동 주기 인텐트 동시 발동 | 추가 데미지/힐 |

#### 9.4.2 시너지 발견 시스템 (검토 중)

```
[개념]
- 시너지는 발견하기 전까지 숨겨짐
- 조건 충족 시 "시너지 발견!" 알림
- 도감에 시너지 목록 등록

[로그라이트 연계]
- 런마다 새로운 시너지 발견 기회
- 영구 진행도로 시너지 도감 확장
```

#### 9.4.3 검토 필요 항목

- 시너지 개수 및 복잡도
- 시너지 밸런스 (필수 조합화 방지)
- 시너지 발견 힌트 제공 여부
- 캐릭터 고유 시너지 vs 범용 시너지
