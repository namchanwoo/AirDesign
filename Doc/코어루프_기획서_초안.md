## 1. 게임 개요

### 1.1 컨셉 요약

**"자동으로 싸우는 유닛들, 전략적으로 개입하는 플레이어"**

유닛들이 **반그리드(Half-Grid)** 위에서 일정 턴 주기마다 자동으로 행동하고, 플레이어는 카드를 통해 전장에 개입하여 전투의 흐름을 바꾸는 턴제 전략 게임.

### 1.2 핵심 키워드

- **반그리드 전투** (진영 분리 시스템)
- 글로벌 모래시계 + 개별 턴 카운터
- 카드 개입 시스템
- **인텐트 전환 전술**
- 타이밍 플레이

### 1.3 레퍼런스 게임

| 게임 | 참고 요소 |
|------|----------|
| **One Step From Eden** | 실시간 그리드 전투, 카드 타이밍 |
| **Alina of the Arena** | 그리드 포지셔닝, 전략적 깊이 |
| **Card-en-Ciel** | 실시간 이동 + 카드 조합 |
| **클래시 로얄** | 실시간 진행 중 카드 개입 감각 |
| **Mega Man Battle Network** | 진영 분리 그리드의 원조 |

### 1.4 핵심 재미 요소

1. **리듬과 개입**: 유닛들의 자동 행동 주기를 읽고 최적의 타이밍에 카드 개입
2. **전선 관리**: 여러 행(전선)을 동시에 관리하며 포지셔닝 결정
3. **인텐트 스위칭**: 같은 캐릭터도 인텐트에 따라 완전히 다른 운용
4. **장전과 발동 분리**: 카드를 미리 준비해두고 결정적 순간에 터뜨리는 쾌감

## 2. 전투 시스템

### 2.1 반그리드 전장 구조

> **상세 내용**: 반그리드_시스템_상세기획서.md 참조

#### 2.1.1 핵심 개념: 진영 분리

전장은 **2개의 독립된 진영 그리드**로 구성된다.

```
전체 전장 (탑뷰 2D - 좌우 배치)
┌─────────────────┬─────────────────┐
│   아군 진영      │      적 진영     │
│  (완전 통제)    │   (읽기 전용)    │
│                 │                 │
│  배치 & 최적화  │   관찰 & 예측   │
│         ── 공격 방향 ->            │
└─────────────────┴─────────────────┘
```

- **아군 진영 (왼쪽)**: 배치하고 최적화하는 영역
- **적 진영 (오른쪽)**: 관찰하고 예측하는 영역
- 서로 상대 진영으로 **절대 침범 불가**

#### 2.1.2 그리드 규격

| 규격 | 아군 진영 | 적 진영 | 적합 상황 |
|------|----------|--------|----------|
| **3x3** | 3열x3행 | 3열x3행 | 일반 전투, 튜토리얼, 초반 |
| **4x4** | 4열x4행 | 4열x4행 | 정예 전투, 보스 전투 |
| **5x5** | 5열x5행 | 5열x5행 | 특수 이벤트, 최종 보스 (예외) |

#### 2.1.3 행(Row)과 열(Column)의 역할

| 축 | 역할 | 전략적 의미 |
|----|------|-------------|
| **행(Row)** | 전선(Battle Line) | 같은 행의 적과 마주봄, 타겟팅 기준 |
| **열(Column)** | 깊이(Depth) | 전방/후방 개념, 피격 우선순위 |

#### 2.1.4 반그리드 시각화 예시

```
[아군 진영]                    [적 진영]
열1   열2   열3   열4    ->    열4   열3   열2   열1
┌────┬────┬────┬────┐      ┌────┬────┬────┬────┐
│ A1 │    │    │    │ 행1 <->│    │    │    │ E1 │ 행1
├────┼────┼────┼────┤      ├────┼────┼────┼────┤
│    │ A2 │    │    │ 행2 <->│    │    │ E2 │    │ 행2
├────┼────┼────┼────┤      ├────┼────┼────┼────┤
│ A3 │ A4 │    │    │ 행3 <->│    │ E4 │    │ E3 │ 행3
├────┼────┼────┼────┤      ├────┼────┼────┼────┤
│    │    │    │    │ 행4 <->│    │    │    │    │ 행4
└────┴────┴────┴────┘      └────┴────┴────┴────┘
(후방)    (전방)            (전방)    (후방)

행1: A1 vs E1 (1:1 대치)
행2: A2 vs E2 (1:1 대치)
행3: A3, A4 vs E3, E4 (2:2 대치)
행4: 빈 전선 (양측 모두 없음)
```

#### 2.1.5 타겟팅 기본 규칙

- **같은 행**에 있는 적만 타겟 가능
- 같은 행에 여러 적이 있으면 **최전방(아군과 가까운) 적** 우선
- 같은 행에 적이 없으면 **대기 행동** 실행

### 2.2 유닛 구성

#### 2.2.1 파티 구성

- **아군 파티**: 최대 4명
- **적 구성**: 스테이지별 상이 (1~다수)

#### 2.2.2 유닛 기본 스탯

| 스탯 | 설명 |
|------|------|
| HP | 체력, 0이 되면 전투 불능 |
| 턴 주기 | 인텐트 행동 실행 간격 (턴 단위) |
| 스피드 | 동시 발동 시 처리 순서 결정 (높을수록 먼저) |
| 기본 데미지 | 공격 인텐트 시 기본 피해량 |
| 코스트 충전량 | 대기 시 충전되는 코스트 양 |

## 3. 턴 시스템

> **상세 내용**: 턴_시스템_상세기획서.md 참조

### 3.1 하이브리드 턴 시스템 개요

```
[Layer 1] 글로벌 모래시계 ──── 모든 유닛이 공유하는 시간 기준 (1턴 = 약 3초)
              │
              v
[Layer 2] 개별 턴 카운터 ───── 유닛마다 다른 행동 주기 ([N] -> [N-1] -> ... -> [0] 발동!)
              │
              v
[Layer 3] 스피드 스탯 ──────── 동시 발동 시 처리 순서
```

### 3.2 핵심 동작

1. **모래시계 뒤집힘** (약 3초마다)
2. 모든 유닛 **턴 카운터 -1**
3. 카운터가 0인 유닛 **행동 발동**
4. 발동 후 카운터 **최대 주기로 리셋**

### 3.3 동시 발동 처리

| 우선순위 | 기준 |
|:--------:|------|
| 1순위 | **스피드** 높은 순서대로 처리 |
| 2순위 | 스피드 동률 시: 아군 우선 |
| 3순위 | 아군 간 동률 시: UI 슬롯 순서 |

### 3.4 턴 조작 효과

| 효과 | 설명 |
|------|------|
| **가속 (Haste)** | 턴 주기 -1 |
| **둔화 (Slow)** | 턴 주기 +1 |
| **즉시 발동** | 카운터 즉시 0으로 |
| **스턴** | 카운터 진행 + 행동 모두 정지 |

## 4. 인텐트 시스템 (Intent System)

> **상세 내용**: 인텐트_시스템_상세기획서.md 참조

### 4.1 기본 개념

각 캐릭터는 **최대 3개의 고유 인텐트 슬롯**을 보유하며, 인텐트에 따라 자동 행동 패턴이 완전히 달라진다.

**핵심 설계 철학**:
- 인텐트는 "공격/방어/지원/충전" 같은 **역할 고정이 아님**
- **고유 인텐트만** (캐릭터 전용, 최대 3개)
- 각 인텐트의 구체적 행동은 **캐릭터마다 완전히 다름**
- 캐릭터의 서사, 정체성, 특성이 인텐트에 반영됨

### 4.2 인텐트 구조

```
캐릭터
├── 고유 인텐트 1: [행동 패턴 A] (초기 해금)
├── 고유 인텐트 2: [행동 패턴 B] (조건 해금)
└── 고유 인텐트 3: [행동 패턴 C] (조건 해금)
```

### 4.3 인텐트 행동 구조

각 인텐트는 **실행 조건**과 **두 가지 행동**을 가진다:

```
[행동 주기 도달]
       │
       v
  ┌───────────┐
  │ 실행 조건 │
  │   체크    │
  └─────┬─────┘
        │
   ┌────┴────┐
   v         v
[조건 충족]  [조건 불충족]
   │              │
   v              v
[실행 행동]  [대기 행동]
(주요 효과)  (보조 효과)
```

### 4.4 인텐트 전환 규칙

| 규칙 | 내용 |
|------|------|
| **비용** | 없음 (자유롭게 전환 가능) |
| **전환 방식** | **즉시 전환**, 해당 턴은 대기 상태 |
| **적용 시점** | **다음 턴부터** 최대 주기로 시작 |
| **이전 인텐트** | 남은 턴 무관하게 즉시 취소 |
| **제한 사항** | 카드 사용 중에는 인텐트 변경 **불가** |

**전략적 의미**: 자주 전환하면 **행동 실행 빈도가 낮아짐** (전환 턴 손실)

## 5. 이동 시스템

> **상세 내용**: 반그리드_시스템_상세기획서.md 참조

### 5.1 핵심 원칙

유닛은 기본적으로 **제자리 유지**. 이동은 **카드 사용**으로만 가능.

| 규칙 | 내용 |
|------|------|
| **이동 거리** | 1칸 (기본) |
| **이동 방향** | 상하좌우 4방향 (대각선 불가) |
| **이동 범위** | 자기 진영 내에서만 |
| **진영 침범** | **절대 불가** |

### 5.2 이동 카드 예시

| 카드명 | 코스트 | 효과 |
|--------|--------|------|
| **전진** | 1 | 아군 1명을 전방(열+1)으로 1칸 이동 |
| **후퇴** | 1 | 아군 1명을 후방(열-1)으로 1칸 이동 |
| **측면 이동** | 1 | 아군 1명을 상하(행+-1)로 1칸 이동 |
| **위치 교환** | 2 | 아군 2명의 위치를 교환 |

### 5.3 이동의 전략적 가치

| 상황 | 이동 가치 |
|------|----------|
| 빈 행 -> 적 있는 행 | 높음 (공격 가능) |
| 위험 행 -> 안전 행 | 높음 (생존) |
| 적 없는 행 유지 | 중간 (코스트 충전) |

## 6. 카드 시스템

> **상세 내용**: 카드_시스템_상세기획서.md 참조

### 6.1 핵심 설계 철학

**"캐릭터 네러티브에서 유래하지만, 전략적 선택은 플레이어에게"**

- 카드는 캐릭터의 스토리와 정체성에서 유래
- **어떤 캐릭터든 어떤 카드든 사용 가능**
- 카드 배치와 운용은 **전략 지휘관(플레이어)**의 판단

### 6.2 파티 공유 덱

**핵심**: 파티 전체가 **하나의 덱**을 공유

```
[파티 공유 덱] -> [파티 핸드] -> [어떤 캐릭터든 슬롯에 배치] -> [발동]
                    ^                    ^                      ^
                 매 턴 1장           AP 소모               타입별 상이
                  드로우
```

| 항목 | 내용 |
|------|------|
| **덱 최대** | 40장 |
| **중복 카드** | 허용 |
| **전략** | 작은 덱 = 빠른 드로우 / 큰 덱 = 다양한 대응 |

### 6.3 드로우 시스템

| 항목 | 내용 |
|------|------|
| **드로우 타이밍** | 매 턴 1장 (테스트 후 2~3장 조정 가능) |
| **핸드 제한** | 5장 (초과 시 드로우 불가) |
| **카드 버리기** | **수동 버리기 예약** (자동 버림 없음) |
| **덱 소진 시** | 묘지 셔플 → 새 덱 구성 |

**턴별 카드 흐름**:
```
[턴 증가] → [버리기 예약 카드 처리] → [1장 드로우] → [인텐트 실행]
                                         ↓
                               [턴 진행 중: 배치/발동/버리기예약]
```

### 6.4 카드 메커니즘 (3종류)

| 메커니즘 | 설명 | 발동 방식 |
|----------|------|-----------|
| **액션 (Action)** | 플레이어가 직접 발동 | 스페이스바 2회 연타 |
| **리액션 (Reaction)** | 조건 충족 시 자동 발동 | 조건부 확정 발동 |
| **인챈트 (Enchant)** | 지속 효과 부여 | 배치 즉시 효과 적용, **제거 시 100% 비용 소모** |

### 6.5 카드 슬롯

| 항목 | 내용 |
|------|------|
| **초기 슬롯** | 모든 캐릭터 1칸 |
| **최대 슬롯** | 4~5칸 (성장/아이템/스토리로 확장) |
| **카드 배치** | **어떤 카드든 어떤 캐릭터 슬롯에든 배치 가능** |

### 6.6 코스트 시스템 (AP)

**확정: 캐릭터별 개인 코스트 (AP)**

```
전사 A: ████░░░░ (4/8 AP)  <- A 슬롯에 배치 시 A의 AP 소모
힐러 B: ██░░░░░░ (2/8 AP)  <- B 슬롯에 배치 시 B의 AP 소모
```

### 6.7 카드 획득

| 경로 | 설명 |
|------|------|
| **캐릭터 네러티브** | 캐릭터 고유 스토리에서 유래 (주요 경로) |
| **보스 드롭** | 보스 처치 시 해당 보스 카드풀에서 랜덤 1장 |
| **상점/스토리** | 구매 또는 스토리 보상 |

### 6.8 카드-캐릭터 시너지

카드는 범용적이지만, **효율 보너스** 존재:

| 메커니즘 | 설명 |
|----------|------|
| **태그 매칭** | 카드/캐릭터 공유 태그에 따라 보너스 |
| **스탯 스케일링** | 카드 효과가 캐릭터 스탯에 비례 |

**핵심**: 도적 카드를 마법사가 사용해도 됨. 효율은 다를 수 있지만 강제 아님.

## 8. 핵심 전략 요소

### 8.1 포지셔닝 전략

> **상세 내용**: 반그리드_시스템_상세기획서.md 참조

**전선(행) 배치의 중요성**:
- 어떤 적과 마주보게 할 것인가
- 빈 행에 배치 -> 대기 상태 -> 안전한 코스트 충전

**깊이(열) 배치의 중요성**:
- 전방(열N): 적 공격에 먼저 노출, 탱커 역할
- 후방(열1): 보호받으며 딜링/지원

### 8.2 인텐트 운용 전략

> **상세 내용**: 인텐트_시스템_상세기획서.md 참조

**기본 원칙**:
- 상황에 따른 인텐트 즉시 전환
- 캐릭터별 "핵심 인텐트" 파악 및 활용
- 인텐트 조합에 따른 팀 시너지

**전환 타이밍 판단**:
- 행동 주기 손실 vs 전술적 이득
- 적 인텐트 발동 전 선제 대응
- 카드 사용과 인텐트 전환 순서 계획

### 8.3 카드 타이밍 전략

**장전 타이밍**:
- 코스트가 충분할 때 미리 장전
- 어떤 캐릭터에게 어떤 카드를 줄 것인가

**발동 타이밍**:
- 적 행동 직전에 끼어들기
- 아군 행동과 연계
- 결정적 순간에 한꺼번에 발동

### 8.4 리듬 읽기

## 부록 A: 용어 정리

> **상세 용어**: 각 상세기획서 부록 참조

| 용어 | 정의 |
|------|------|
| **반그리드 (Half-Grid)** | 전장의 절반만 직접 통제하는 진영 분리 시스템 |
| **진영 (Side)** | 아군/적군이 점유하는 독립된 그리드 영역 |
| **행(Row) / 전선** | 그리드의 가로줄, 타겟팅의 기준 |
| **열(Column) / 깊이** | 그리드의 세로줄, 전후방 개념 |
| **인텐트 (Intent)** | 캐릭터의 자동 행동 의지를 정의하는 단위 |
| **고유 인텐트** | 특정 캐릭터 전용 인텐트 (최대 3개) |
| **인텐트 전환** | 현재 인텐트를 다른 인텐트로 즉시 변경 (주기 리셋) |
| **턴 (Turn)** | 글로벌 모래시계가 뒤집어지는 단위 (약 3초) |
| **턴 주기** | 유닛이 자동 행동을 실행하는 간격 (턴 단위) |
| **턴 카운터** | 현재 남은 턴 수 (0이 되면 행동 발동) |
| **글로벌 모래시계** | 화면 중앙의 공용 타이머, 1턴의 기준 |
| **스피드** | 동시 발동 시 처리 순서 결정 (높을수록 먼저) |
| **실행 행동** | 조건 충족 시 실행되는 주요 행동 |
| **대기 행동** | 조건 불충족 시 실행되는 보조 행동 |
| **코스트** | 카드 장전에 필요한 자원 |
| **장전** | 핸드의 카드를 캐릭터 슬롯에 배치하는 행위 |
| **발동** | 장전된 카드를 실제로 사용하는 행위 |
| **핸드** | 현재 손에 들고 있는 카드들 |
| **덱** | 전투에 가져가는 카드 묶음 |
| **슬롯** | 캐릭터에게 카드를 장전할 수 있는 칸 |
